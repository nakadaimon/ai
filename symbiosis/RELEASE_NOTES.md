# Release Notes — 4.0.0

Symbiosis 4.0 is a consolidation release. No new concepts — the 3.0 architecture held up in the field. What 4.0 does is promote what proved itself, slim what grew, and measure what was previously taken on faith. Three words: promote, slim, measure.

**Promote.** The harness gains its third file: `hooks.md`. Commands are triggers you invoke; hooks are deterministic if-X-then-Y rules the agent fires on its own when it observes a state — a session-end phrase, a task slipping by in conversation, a derived index gone stale behind its source. The pattern ran in a live deployment before it shipped here, which is how reference changes should arrive: field-proven, then generalized. Each hook carries a detection surface (phrase, filesystem, session signal) so you know what your runtime supports, and `lint hooks` keeps the table honest. Two more field promotions ride along: the session scratchpad is now created at session start — so a crash always leaves a resume point — and the procedural domain gains three opt-in folders (`recipes/`, `prompts/`, `schematics/`) for delegation briefs, prompt templates, and build blueprints.

**Slim.** The eager-loaded boot stack now has a budget: ~5,000 words, enforced by `lint bootstrap`. Command entries stay at trigger + outcome; implementation detail moves to a skill or script in `memento/procedural/` that the entry points to. Every word in the harness is paid for at every session start — now the bill is visible.

**Measure.** Measurement used to stop at the manifesto (the blind onboarding tests); the running harness itself was never measured. 4.0 extends the culture inward: `lint subtraction` gains ablation light (run a small eval deck with and without a questioned rule; no delta means no keep) and a boot metric recorded at every release. `ingest status` gives the multiplying capture points an aggregate health signal — one cheap pass that reports unprocessed entries across every inbox, instrumenting what was previously an exit criterion you'd only notice too late. And a cited-recall principle closes the retrieval loop: answers from the memory layer cite their source file, and absence is stated rather than improvised.

One addition is pure pedagogy: the README now maps symbiosis onto the three jobs every memory system must do — storage, injection, recall — so you can compare it against other frameworks on even terms. `ingest` is storage. The boot read order is injection. The retrieval levels are recall. Most frameworks solve one or two; the structure exists so the three can share a substrate.

Upgrading from 3.0 is mechanical: add `hooks.md` (start from the three generic starters), slot it into the read order after `commands.md`, adopt the boot budget, and move any command spec that outgrew its entry into `memento/procedural/`. Nothing else breaks. The major bump is for the read-order and tree changes, not for migration pain.
